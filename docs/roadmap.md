## Как эффективно работать в Cursor

### 1. Открывай проект целиком, не файл

Cursor видит всю кодобазу через индексацию. Открывай `visa_kit/` как папку,
не отдельные файлы. Тогда AI может ссылаться на любой файл.

### 2. Начинай каждую сессию с CLAUDE.md

Когда начинаешь новую задачу, в чате Cursor пиши:
"Прочитай CLAUDE.md в корне и в backend/app/<релевантная папка>.
Я хочу <твоя задача>."

Это сэкономит часы — AI сразу поймёт контекст и соглашения.

### 3. Используй существующий код как образец

"Создай api/positions.py по образцу api/companies.py" — проще для AI и точнее
по результату, чем "напиши CRUD для Position с нуля".

### 4. Тестируй после каждого изменения

После добавления нового файла:
```bash
cd backend && PYTHONPATH=. python3 -c "from app.api.positions import router; print('OK')"
```

После добавления модели — миграция и проверка БД:
```bash
alembic revision --autogenerate -m "..."
alembic upgrade head
```

### 5. Декомпозиция на маленькие задачи

Не "сделай весь админский фронт". А "сделай страницу со списком заявок,
с фильтрами по статусу и поиском по имени". Маленькие шаги — меньше ошибок.

### 6. Когда что-то ломается — спроси Claude

"У меня вот такая ошибка: <вставь stack trace>. Я делал X, ожидал Y. Что
не так?" Это работает в 9/10 случаев.

## Чего избегать

- **Не давай AI писать миграции автоматом без проверки.** Alembic иногда
  пропускает важные вещи (например, переименование колонок видит как
  drop+create, что теряет данные). Всегда читай миграцию глазами перед
  `alembic upgrade head`.

- **Не пиши raw SQL.** SQLModel/SQLAlchemy решают всё что нужно. Если
  кажется что нужен raw SQL — скорее всего, ты упускаешь возможность ORM.

- **Не храни секреты в репе.** `.env` — в `.gitignore`. Раздавай через
  `.env.example`.

- **Не делай "красивые" компоненты с нуля.** Бери готовые из shadcn/ui:
  `npx shadcn@latest add button card form table dialog`. Это сэкономит
  дни работы и выглядит профессионально.
