### Phase 2 — Максимальная автоматизация (по приоритету окупаемости)

**Pack 10 — OCR паспорта и автозаполнение анкеты** (3-5 дней работы)
- Клиент загружает скан паспорта → OCR + ИИ извлекает все поля
- Автозаполнение фамилии/имени, дат, номера паспорта, места выдачи
- Поддержка российского, азербайджанского, казахского и других паспортов
- Технологии: Tesseract или Claude Vision API + structured output
- Окупаемость: огромная, клиенты ненавидят вводить данные руками + меньше опечаток
- Расходы: ~$0.05-0.10 за один паспорт через Claude Vision

**Pack 11 — Авто-проверка пакета перед скачиванием** (2-3 дня)
- ИИ-агент за 30 секунд читает все 10 сгенерированных документов
- Проверяет: согласованность дат, совпадение сумм в актах и счетах, орфография в ФИО, склонения, формат паспорта
- Возвращает «всё ок» или конкретный список несоответствий
- Окупаемость: одна избегнутая ошибка = одобрение визы клиенту
- Расходы: ~$0.10 за пакет

**Pack 12 — Email-агент для коммуникации с клиентом** (5-7 дней)
- Автоответчик на типовые вопросы: «когда подача?», «получили мои документы?», «нужно ли мне сейчас что-то делать?»
- Notify-уведомления клиентам: «вы заполнили 80% анкеты, осталось загрузить паспорт»
- Напоминания о сроках, дедлайнах
- ОБЯЗАТЕЛЬНО: модерация менеджером перед отправкой сложных писем
- Окупаемость: освобождает 2-3 часа менеджера в день
- Технологии: Claude API + email integration (SMTP/SendGrid)

**Pack 13 — Авто-распределение по компаниям** (2 дня)
- Балансировка нагрузки между 8 компаниями (какая отправила меньше за месяц)
- Учёт географии клиента (Барселона vs Мадрид → выбор представителя)
- Учёт пересечений: чтобы не назначать одного представителя сильно подряд
- Менеджер видит результат и может изменить
- Окупаемость: убирает один шаг ручного распределения

**Pack 14 — Перевод-черновик на испанский** (1-2 дня)
- Claude переводит документы на испанский на уровне присяжного переводчика
- Это ЧЕРНОВИК — финальную проверку и подпись делает живой переводчик (юридическое требование UGE)
- Сократит время на этапе «у переводчика» с 3-5 дней до 1 дня
- Окупаемость: ускорение pipeline для клиента

**Pack 15 — Мониторинг статусов в UGE** (хрупкое, нужно тестировать)
- Browser-agent через Claude Computer Use периодически открывает портал UGE
- Проверяет статусы поданных заявок, апдейтит БД
- Уведомляет менеджера о изменениях
- Риски: UGE может ломать UI, блокировать IP
- ВАЖНО: не для подачи документов — только для чтения статусов

---

### Что НЕ автоматизируем (намеренно)

1. **Итоговое решение менеджера перед подачей** — юридическая ответственность лежит на агентстве. Перед каждой подачей человек должен глазами просмотреть пакет
2. **Личная коммуникация в сложных кейсах** (развод с детьми, отказы в прошлом, нестандартные ситуации) — ИИ помогает готовить варианты, но решение и эмпатия за человеком
3. **Оценка рисков отказа** в нестандартных кейсах — требует юридической экспертизы
4. **Подача документов в UGE** — UGE не имеет API, серая зона
5. **Подписание документов** (ЭП) — юридическое действие, ответственность не передать

---

### Цели автоматизации

**После всех пакетов выше:**
- Время менеджера на одну заявку: с 4-6 часов → до 30 минут
- Один менеджер может вести: с 50 → до 200+ заявок/мес
- Возможные сценарии:
  - Рост выручки в 4-6 раз без увеличения штата
  - ИЛИ снижение цены для клиента в 2-3 раза при сохранении маржи
  - ИЛИ комбинация (умеренный рост штата + умеренное снижение цены + рост маржи)

## Технические детали

- Windows, Python 3.14, Node 24
- Проект: `D:\VISA\visa_kit\`
- Backend: FastAPI + SQLite + SQLModel
- Frontend: Next.js 15 + Tailwind
- БД: SQLite (`visa_kit.db` в backend/)
- LLM: Claude Sonnet (для рекомендаций)

## Запуск

```powershell
# Backend
cd D:\VISA\visa_kit\backend
.venv\Scripts\Activate.ps1
uvicorn app.main:app --reload

# Frontend (другое окно)
cd D:\VISA\visa_kit\frontend
npm run dev
```

Логин в админку: `admin@visa-kit.local` (без пароля в dev).
